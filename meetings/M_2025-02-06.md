- Wrote a benchmark.sh:
    - Parameters:
        1. election type
        2. Number of bits
        3. Number of Votes/Cand
        4. Election type specific parameters
    - Steps:
        1. Generate a circom test file (basically only includes the main component with assert<ElectionType> template)
        2. Generate a sage test file which outputs a ballot in <electionType> with the specified type and dimensions as well as all the other inputs (randomnesses, generator, public key, encrypted ballot) for the generated circom test file (in json format).
        3. Compile the generated circom test file and generatte the witness with the given input (using the genCircom.sh script I wrote earlier)
        4. Prepares the proof by generating a zkey file from a provided powersOfTau file
            - Uses a new script prepareProof.sh
            - Currently makes one very much not random contribution to the zkey file (I also implemented the option of using a random beacon but I have not tested this yet.)
            - Question: Can I remove the contribution to the zkey file? (This takes up a lot of time in the proof preparation)
        5. Create the proof using a new script createProof.sh
            - This basically just calls the snarkjs groth16 prove command
        6. Varifies the proof using a new script verifyProof.sh
            - This basically just calls the snarkjs groth16 verify command
        7. Export the constraints and proof times to two csv files:
            1. times/<electionType>.csv. Each row has
                - The input parameters of the benchmark.sh script as an identifier (except for the electionType)
                - proof preparation, creation and verification times
            2. constraints/<electionType>.csv Each row has
                - The input parameters of the benchmark.sh script as an identifier (except for the electionType)
                - non-linear, linear and total constraints
